# 海星模型（Starfish Retrospective）

## 简介

海星模型是一种敏捷回顾方法，从五个维度反思行为策略，目标是"下次更高效"。

## 五个维度

| 维度 | 含义 | 问法 |
|------|------|------|
| ✋ 保持（Keep） | 有效、值得继续的做法 | 这次哪些做法带来了价值？ |
| 🚀 开始（Start） | 应该做但没做的 | 下次应该增加什么新的做法？ |
| 🛑 停止（Stop） | 带来负面效果或浪费的行为 | 哪些行为应该立刻停止？ |
| ⬆️ 更多（More） | 有效但量不够的行为 | 哪些有效行为可以做得更多？ |
| ⬇️ 更少（Less） | 有价值但成本过高的行为 | 哪些行为可以做但不需要那么多？ |

## 使用原则

1. **聚焦行为**，不评判意图。写具体行为，不写"态度"
2. **每格至少 1 条**，避免空置维度
3. **优先写 Start 和 Stop**，这两项改变价值最高
4. **不搞形式主义**：如果某维度确实没有可写内容，写"无（暂无发现）"

## 示例

### ✋ 保持（Keep）
- 使用 `search_file` 先定位文件再读取，避免路径猜测

### 🚀 开始（Start）
- 部署前先用 `--help` 确认 CLI 参数，不依赖记忆

### 🛑 停止（Stop）
- 不串联 `git status && git add -A && git commit`，容易误提交

### ⬆️ 更多（More）
- 多在执行前确认文件结构，减少后期回头修复

### ⬇️ 更少（Less）
- 减少"尝试性"命令（先猜参数再报错），改为先查文档
