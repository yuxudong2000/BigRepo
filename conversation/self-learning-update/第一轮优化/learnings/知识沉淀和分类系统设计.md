#### 2.1 三层记忆架构

**层级 1：内置记忆（Built-in Memory）**
- **MEMORY.md**：Agent 个人笔记（环境事实、项目约定、工具陷阱）

#### 3.2 写并发控制

**WAL 模式 + 应用层重试**：
```python
# hermes_state.py
_WRITE_MAX_RETRIES = 15
_WRITE_RETRY_MIN_S = 0.020  # 20ms
_WRITE_RETRY_MAX_S = 0.150  # 150ms

def _execute_write(fn: Callable) -> T:
    for attempt in range(_WRITE_MAX_RETRIES):
        try:
            with self._lock:
                self._conn.execute("BEGIN IMMEDIATE")  # 立即获取写锁
                result = fn(self._conn)
                self._conn.commit()
                return result
        except sqlite3.OperationalError as exc:
            if "locked" in str(exc).lower():
                # 随机抖动重试（打破确定性回退的护航效应）
                time.sleep(random.uniform(_WRITE_RETRY_MIN_S, _WRITE_RETRY_MAX_S))
                continue
            raise
```

**关键设计决策**：
- ❌ **不使用 SQLite 内置 busy handler**（确定性睡眠导致护航效应）
- ✅ **应用层随机抖动**（自然错开竞争写入）
- ✅ **PASSIVE checkpoint 每 50 次写入**（防止 WAL 无限增长）

### 4. 知识图谱的可能性

#### 4.1 现有图结构实现

**Hindsight Provider 的知识图谱**：
- **实体解析**：从对话中提取命名实体
- **关系建模**：实体间的语义关系

**限制**：
- 依赖外部服务（Hindsight Cloud API）
- 图结构对用户不可见（黑盒）

- 用途：追踪会话血缘，排除当前会话

### 5. 向量化检索的集成方案

#### 5.1 现有向量化实现

**Holographic Memory Provider**（<kfile name="holographic/retrieval.py" path="plugins/memory/holographic/retrieval.py">holographic/retrieval.py</kfile>）：

```python
# HRR (Holographic Reduced Representations) 向量
class HRRVectorizer:
    def encode_text(text: str, dim: int = 1024) -> np.ndarray:
        """编码为相位向量（复数单位圆）"""
        
    def similarity(vec1: np.ndarray, vec2: np.ndarray) -> float:
        """余弦相似度 [-1, 1]"""
        
# 混合检索管道
def search(query: str, category: str, min_trust: float, limit: int):
    # 1. FTS5 候选召回（limit * 3）
    candidates = fts_candidates(query, category, limit * 3)
    
    # 2. Jaccard 重排
    query_tokens = tokenize(query)
    for fact in candidates:
        content_tokens = tokenize(fact.content)
        jaccard = len(query_tokens & content_tokens) / len(query_tokens | content_tokens)
        
    # 3. HRR 向量相似度
    if self.hrr_weight > 0:
        fact_vec = hrr.bytes_to_phases(fact.hrr_vector)
        query_vec = hrr.encode_text(query, self.hrr_dim)
        hrr_sim = hrr.similarity(query_vec, fact_vec)
        
    # 4. 融合评分
    relevance = (fts_weight * fts_score 
                 + jaccard_weight * jaccard 
                 + hrr_weight * hrr_sim)
    score = relevance * trust_score * temporal_decay
```

**Hindsight Provider 的向量检索**：
- 由 Hindsight Cloud API 提供
- 细节不透明（黑盒）
- 支持语义搜索 + 关键词混合

#### 5.2 推荐的向量化架构

**设计原则**：
1. **模型无关**：支持多种向量模型（OpenAI, Sentence-Transformers, 本地）
2. **存储分离**：向量与元数据分离存储
3. **增量索引**：支持在线更新，无需全量重建
4. **混合检索**：结合关键词 + 向量 + 图遍历

**架构设计**：

```python
# 向量存储抽象
class VectorStore(ABC):
    @abstractmethod
    def upsert(self, vectors: List[Vector]) -> None:
        """插入或更新向量"""
        
    @abstractmethod
    def search(self, 
               query_vector: np.ndarray,
               top_k: int = 10,
               filters: Dict[str, Any] = None) -> List[SearchResult]:
        """向量检索"""
        
    @abstractmethod
    def delete(self, ids: List[str]) -> None:
        """删除向量"""

# 具体实现
class SQLiteVectorStore(VectorStore):
    """基于 sqlite-vss 或 sqlite-vec 的本地向量存储"""
    
class ChromaVectorStore(VectorStore):
    """Chroma 向量数据库（已有技能支持）"""
    
class QdrantVectorStore(VectorStore):
    """Qdrant 向量数据库（已有技能支持）"""

# 向量化管理器
class VectorizationManager:
    def __init__(self, 
                 vectorizer: TextVectorizer,
                 store: VectorStore,
                 hybrid_mode: str = "rrf"):  # reciprocal rank fusion
        self.vectorizer = vectorizer
        self.store = store
        self.hybrid_mode = hybrid_mode
        
    def index_memory(self, memory_entries: List[MemoryEntry]):
        """向量化记忆条目"""
        vectors = [
            Vector(
                id=entry.id,
                vector=self.vectorizer.encode(entry.content),
                metadata={
                    "source": entry.source,  # memory/user/session
                    "timestamp": entry.timestamp,
                    "tags": entry.tags,
                }
            )
            for entry in memory_entries
        ]
        self.store.upsert(vectors)
        
    def hybrid_search(self, 
                      query: str,
                      keyword_results: List[SearchResult],
                      top_k: int = 10) -> List[SearchResult]:
        """混合检索：关键词 + 向量"""
        # 1. 向量检索
        query_vec = self.vectorizer.encode(query)
        vector_results = self.store.search(query_vec, top_k * 2)
        
        # 2. 融合排序（RRF）
        if self.hybrid_mode == "rrf":
            return reciprocal_rank_fusion(keyword_results, vector_results, k=60)
        elif self.hybrid_mode == "weighted":
            return weighted_fusion(keyword_results, vector_results, alpha=0.5)
```

**SQL 表设计（sqlite-vec 方案）**：
```sql
-- 向量表

-- 向量检索示例
SELECT

    vm.source,
    vec_distance_L2(mv.embedding, ?) as distance
FROM memory_vectors mv
JOIN vector_metadata vm ON vm.id = mv.id

ORDER BY distance ASC
LIMIT 10;
```

```python
class MultiModalRetriever:
    def __init__(self):
        self.keyword_index = FTS5Index()
        self.vector_index = VectorStore()
        self.graph_index = KnowledgeGraph()
        
    def search(self, 
               query: str,
               mode: str = "auto",  # auto/keyword/vector/graph/hybrid
               filters: Dict = None,
               top_k: int = 10) -> List[KnowledgeEntry]:
        """统一检索接口"""
        
        if mode == "auto":
            mode = self._detect_mode(query)
            
        if mode == "keyword":
            return self.keyword_search(query, filters, top_k)
        elif mode == "vector":
            return self.vector_search(query, filters, top_k)
        elif mode == "graph":
            return self.graph_search(query, filters, top_k)
        elif mode == "hybrid":
            # RRF 融合多种检索结果
            keyword_results = self.keyword_search(query, filters, top_k * 2)
            vector_results = self.vector_search(query, filters, top_k * 2)
            graph_results = self.graph_search(query, filters, top_k * 2)
            return self._fuse_results([keyword_results, vector_results, graph_results], top_k)
            
    def _detect_mode(self, query: str) -> str:
        """启发式检测最佳检索模式"""
        # 精确匹配 → keyword
        if re.search(r'["\'`]', query):
            return "keyword"
        # 实体名称 → graph
        if self._contains_entity_name(query):
            return "graph"
        # 语义问题 → vector
        return "vector"
```

**2. 批量向量化**：
```python
class BatchVectorizer:
    def __init__(self, model, batch_size: int = 32):
        self.model = model
        self.batch_size = batch_size
        self.queue: List[str] = []
        self.pending_updates: Dict[str, Future] = {}
        
    def encode_async(self, text: str, entry_id: str) -> Future:
        """异步批量编码"""
        future = Future()
        self.queue.append(text)
        self.pending_updates[entry_id] = future
        
        if len(self.queue) >= self.batch_size:
            self._flush_batch()
            
        return future
        
    def _flush_batch(self):
        if not self.queue:
            return
        vectors = self.model.encode(self.queue)
        for entry_id, vector in zip(self.pending_updates.keys(), vectors):
            self.pending_updates[entry_id].set_result(vector)
        self.queue.clear()
        self.pending_updates.clear()
```

class MemoryVersion:
    """记忆条目的版本历史"""
    id: str
    entry_id: str
    content: str
    version: int
    created_at: float
    created_by: str  # session_id

class VersionedMemoryStore(MemoryStore):
    def add(self, target: str, content: str, session_id: str) -> Dict:
        # 添加新条目时记录版本
        entry_id = self._generate_entry_id()
        version = MemoryVersion(
            id=self._generate_version_id(),
            entry_id=entry_id,
            content=content,
            version=1,
            created_at=time.time(),
            created_by=session_id,
            change_type="add"
        )
        self._save_version(version)
        return super().add(target, content)
        
    def get_history(self, entry_id: str) -> List[MemoryVersion]:
        """获取条目的修改历史"""
        return self._load_versions(entry_id)
        
    def rollback(self, entry_id: str, to_version: int) -> Dict:
        """回滚到指定版本"""
        version = self._load_version(entry_id, to_version)
        return self.replace(version.target, version.content, version.content)

class MemorySyncManager:
    """记忆条目跨设备同步"""
    
    def __init__(self, sync_backend: str = "git"):
        # 支持 git/rsync/cloud
        self.backend = self._init_backend(sync_backend)
        
    def push(self):
        """推送本地变更到远程"""
        changes = self._collect_changes()
        self.backend.commit(changes)
        self.backend.push()
        
    def pull(self):
        """从远程拉取变更"""
        remote_changes = self.backend.pull()
        conflicts = self._detect_conflicts(remote_changes)
        
        if conflicts:
            return self._resolve_conflicts(conflicts)
            
        self._apply_changes(remote_changes)
        
    def auto_sync(self, interval: int = 300):
        """后台自动同步（每5分钟）"""
        while True:
            try:
                self.pull()
                self.push()
            except Exception as e:
                logger.error("Auto-sync failed: %s", e)
            time.sleep(interval)

## 四、关键技术挑战

### 1. 性能与可扩展性

**挑战**：
- FTS5 在大规模数据（>100万条消息）下的性能
- 向量检索的内存占用（100K条目 × 384维 ≈ 150MB）
- 知识图谱遍历的时间复杂度

**解决方案**：
- 分片索引（按时间/会话/类型）
- 量化向量（FP32 → INT8）降低存储
- 图缓存热点路径

### 2. 知识一致性

**挑战**：
- 跨数据源的事实冲突（MEMORY.md vs Hindsight）
- 时间敏感信息的过期处理
- 用户偏好的动态变化

**解决方案**：
- 引入置信度评分和溯源链
- 时间戳 + 过期策略
- 增量更新 + 冲突检测

### 3. 隐私与安全

**挑战**：
- 敏感信息泄露（API密钥、个人信息）
- Prompt注入攻击（通过记忆污染系统提示）
- 跨会话信息隔离

**解决方案**：
- 已有的 `_scan_memory_content()` 机制
- PII检测 + 自动脱敏
- 基于会话ID的访问控制

## 五、总结

Hermes 现有的知识管理系统已经具备了完整的**三层架构**（内置记忆 + 插件化Provider + 会话搜索），但在**知识分类、图谱关联、向量检索**方面仍有较大改进空间。

**核心改进方向**：
1. **统一本体层**：建立跨数据源的统一分类体系
2. **知识图谱**：从线性记忆到关系网络
3. **混合检索**：关键词 + 向量 + 图遍历的深度融合
4. **自动化抽取**：从对话中持续学习和更新知识

**关键设计原则**：
- **渐进式迁移**：不破坏现有API和数据格式
- **模块化**：各层独立可替换（向量模型、图存储）
- **性能优先**：批量处理、增量索引、缓存优化
- **隐私保护**：本地优先、脱敏机制、访问控制

通过这些改进，Hermes 可以从"对话助手"进化为真正的"个人知识助理"，实现从检索到推理的跨越。
