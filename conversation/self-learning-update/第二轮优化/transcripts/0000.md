### 1️⃣ **核心 Agent 循环** (`run_agent.py`)

这是整个项目的心脏，约 **12,000 行代码**：

```python
class AIAgent:
    def run_conversation(self, user_message: str) -> dict:
        """核心对话循环（简化版）"""
        while (api_call_count < max_iterations and budget.remaining > 0) \
                or self._budget_grace_call:
            if self._interrupt_requested: break
            
            # 1. 调用 LLM
            response = client.chat.completions.create(
                model=model, 
                messages=messages, 
                tools=tool_schemas
            )
            
            # 2. 如果有工具调用
            if response.tool_calls:
                for tool_call in response.tool_calls:
                    result = handle_function_call(
                        tool_call.name, 
                        tool_call.args, 
                        task_id
                    )
                    messages.append(tool_result_message(result))
                api_call_count += 1
            else:
                # 3. 没有工具调用，返回最终回复
                return response.content
```

**关键特性**：
- **预算控制**：支持多种成本控制策略（token 限制、金额上限）
- **中断机制**：用户可随时打断当前任务
- **提示词缓存**：通过缓存系统提示词降低 API 成本
- **子 Agent 支持**：可递归生成子 Agent 处理并行任务

1. **Why 链深度**: 3-5 层（根据问题复杂度动态调整）
2. **根因分析触发**: 优先 insights，可选 solutions/patterns
3. **原则提炼时机**: 初期手动触发，后期自动（累积 ≥ 10 条根因）
4. **并发策略**: Step 3 并行提取 + Step 3.5 串行分析（深度推理不适合并行）
5. **质量门槛**: Why 链至少 3 层，根因必须指向系统性问题，原则至少 3 个适用场景

### 📋 关键设计决策

1. **Why 链深度**: 3-5 层（根据问题复杂度动态调整）
2. **根因分析触发**: 优先 insights，可选 solutions/patterns
3. **原则提炼时机**: 初期手动触发，后期自动（累积 ≥ 10 条根因）
4. **并发策略**: Step 3 并行提取 + Step 3.5 串行分析（深度推理不适合并行）
5. **质量门槛**: Why 链至少 3 层，根因必须指向系统性问题，原则至少 3 个适用场景
