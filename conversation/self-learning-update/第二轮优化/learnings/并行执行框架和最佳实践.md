# Hermes-Agent 并行执行能力深度研究报告

## 一、核心并行机制概览

### 1.1 三大并行执行层级

Hermes-Agent 的并行架构分为三个层级：

```
┌─────────────────────────────────────────────────────────┐
│  Layer 1: Multi-Tool Concurrent Execution              │
│  (run_agent.py - ThreadPoolExecutor)                   │
│  同时执行多个工具调用                                     │
└────────────────┬────────────────────────────────────────┘
                 │
┌────────────────▼────────────────────────────────────────┐
│  Layer 2: Subagent Delegation                          │
│  (tools/delegate_tool.py - ThreadPoolExecutor)         │
│  并行运行多个子 Agent                                     │
└────────────────┬────────────────────────────────────────┘
                 │
┌────────────────▼────────────────────────────────────────┐
│  Layer 3: Async Tool Operations                        │
│  (session_search, MCP, etc. - asyncio.gather)          │
│  单个工具内的异步 I/O 并行                                │
└─────────────────────────────────────────────────────────┘
```

### 3.1 智能并行调度

```python
def _should_parallelize_tool_batch(tool_calls):
    # 1. 单个工具 → 顺序执行
    if len(tool_calls) <= 1:
        return False
    
    # 2. 包含交互式工具 (clarify) → 顺序执行
    if any(tc.function.name in _INTERACTIVE_TOOLS for tc in tool_calls):

    if file_ops and _has_path_overlap(file_ops):
        return False

    return True
```

### 3.2 并行执行实现

**工作线程池**：
```python
_MAX_TOOL_WORKERS = 10  # 最大并发工具数

def _execute_tool_calls_concurrent(self, ...):
    max_workers = min(num_tools, _MAX_TOOL_WORKERS)
    with concurrent.futures.ThreadPoolExecutor(max_workers=max_workers) as executor:
        futures = []
        for i, (tc, name, args) in enumerate(parsed_calls):
            f = executor.submit(_run_tool, i, tc, name, args)
            futures.append(f)
        
        # 轮询 + 心跳 + 中断检测
        while True:
            done, not_done = concurrent.futures.wait(
                futures, timeout=5.0
            )
            if not not_done:
                break
            
            # 中断检测
            if self._interrupt_requested:
                for f in not_done:
                    f.cancel()
                break
            
            # 心跳（防止网关超时）
            if elapsed % 30 < 6:
                self._touch_activity(f"concurrent tools running ({elapsed}s, ...)")
```

### 3.3 工作线程管理

**线程注册机制**：
```python
def _run_tool(index, tool_call, function_name, function_args):
    # 注册线程 ID（用于中断传播）
    _worker_tid = threading.current_thread().ident
    with self._tool_worker_threads_lock:
        self._tool_worker_threads.add(_worker_tid)
    
    # 设置活动回调（使终端命令能发送心跳）
    from tools.environments.base import set_activity_callback

    finally:
        # 清理线程注册

            self._tool_worker_threads.discard(_worker_tid)
```

## 五、_run_async: 同步-异步桥接

### 5.1 多上下文感知

**核心函数**: <ksymbol name="_run_async" filename="model_tools.py" path="model_tools.py" lines="82-132" type="function">_run_async</ksymbol>

**三种执行路径**：
```python
def _run_async(coro):
    try:
        loop = asyncio.get_running_loop()
    except RuntimeError:
        loop = None
    
    # 路径 1: 已有运行中的事件循环 (Gateway/RL 环境)
    if loop and loop.is_running():
        import concurrent.futures
        pool = concurrent.futures.ThreadPoolExecutor(max_workers=1)
        future = pool.submit(asyncio.run, coro)
        try:
            return future.result(timeout=300)
        finally:
            pool.shutdown(wait=False, cancel_futures=True)
    
    # 路径 2: 工作线程（并行工具执行）
    if threading.current_thread() is not threading.main_thread():
        worker_loop = _get_worker_loop()  # 每线程持久化 loop
        return worker_loop.run_until_complete(coro)
    
    # 路径 3: 主线程（CLI 模式）
    tool_loop = _get_tool_loop()  # 进程级持久化 loop
    return tool_loop.run_until_complete(coro)
```

### 5.2 Event Loop 管理策略

**持久化 Loop 的必要性**：
```python
# 每线程 Loop 缓存
_worker_loops: Dict[int, asyncio.AbstractEventLoop] = {}

def _get_worker_loop():
    tid = threading.current_thread().ident
    if tid not in _worker_loops:
        loop = asyncio.new_event_loop()
        asyncio.set_event_loop(loop)
        _worker_loops[tid] = loop
    return _worker_loops[tid]
```

**优势**：
- 避免 `asyncio.run()` 的创建-销毁开销
- 防止缓存的 `AsyncOpenAI`/`httpx` 客户端出现 "Event loop is closed" 错误
- 工作线程使用独立 Loop，避免与主线程竞争

## 七、ThreadPoolExecutor 使用模式总结

### 7.1 Hermes 中的三种使用场景

| 场景 | 位置 | max_workers | 特点 |
|------|------|-------------|------|
| 并行工具执行 | `run_agent.py` | `min(num_tools, 10)` | 动态调整，最多 10 |
| 子 Agent 委托 | `delegate_tool.py` | `max_concurrent_children` (默认 3) | 配置化，无上限 |
| 单线程异步桥接 | `model_tools.py` | `1` | 临时创建，立即清理 |

### 7.2 最佳实践

#### 7.2.1 上下文管理器

```python
# ✅ 正确：自动清理
with ThreadPoolExecutor(max_workers=3) as executor:
    futures = [executor.submit(work, i) for i in range(10)]
    results = [f.result() for f in futures]

# ❌ 错误：可能泄漏线程
executor = ThreadPoolExecutor(max_workers=3)
# ... 忘记调用 executor.shutdown()
```

#### 7.2.2 中断友好的轮询

```python
# ✅ Hermes 模式：短超时轮询
while True:
    done, pending = wait(futures, timeout=0.5, return_when=FIRST_COMPLETED)
    if not pending:
        break
    if interrupt_requested:
        for f in pending:
            f.cancel()
        break

# ❌ 阻塞模式：无法响应中断
for future in as_completed(futures):
    result = future.result()  # 阻塞直到所有完成
```

#### 7.2.3 异常处理

```python
# ✅ Hermes 模式：每个 future 单独捕获
for future in futures:
    try:
        result = future.result()
    except Exception as e:
        logger.error("Task failed: %s", e)
        result = None  # 降级处理

# ❌ 一个失败导致全部失败
results = [f.result() for f in futures]  # 第一个异常终止循环
```

### 8.3 混合模式的陷阱与解决方案

#### 8.3.1 Event Loop 冲突

**问题**：
```python
# ❌ 在已有 loop 的上下文调用 asyncio.run()
async def outer():
    result = asyncio.run(inner())  # RuntimeError: asyncio.run() cannot be called from a running event loop
```

**Hermes 的解决方案**：
```python
# ✅ _run_async 自动检测上下文
def safe_async_call(coro):
    try:
        loop = asyncio.get_running_loop()
    except RuntimeError:
        return asyncio.run(coro)
    
    # 已有 loop → 使用独立线程
    pool = ThreadPoolExecutor(max_workers=1)
    return pool.submit(asyncio.run, coro).result()
```

#### 8.3.2 缓存客户端的 Loop 绑定

**问题**：
```python
# ❌ 客户端绑定到临时 loop
async def fetch_data():
    client = AsyncOpenAI()  # 绑定到当前 loop
    return await client.chat.completions.create(...)

# 临时 loop 被销毁
asyncio.run(fetch_data())
# 下次调用时，client 仍然引用已关闭的 loop
```

**Hermes 的解决方案**：
```python
# ✅ 持久化 loop
_tool_loop: Optional[asyncio.AbstractEventLoop] = None

def _get_tool_loop():
    global _tool_loop
    if _tool_loop is None or _tool_loop.is_closed():
        _tool_loop = asyncio.new_event_loop()
        asyncio.set_event_loop(_tool_loop)
    return _tool_loop

# 客户端始终绑定到同一个持久 loop
def _run_async(coro):
    if threading.current_thread() is threading.main_thread():
        return _get_tool_loop().run_until_complete(coro)
```

## 九、self-learning skill 的并行优化技术方案

### 9.1 现状分析

假设 self-learning skill 需要：
1. 从多个源并行抓取学习材料
2. 并行总结多个文档
3. 与向量数据库交互
4. 生成学习笔记

### 9.2 推荐架构

#### 9.2.1 方案 A: 混合模式（推荐）

```python
class SelfLearningSkill:
    def __init__(self):
        self.http_client = None  # 延迟初始化 async client
    
    async def _fetch_materials(self, sources: List[str]):
        """异步抓取学习材料"""
        if self.http_client is None:
            self.http_client = httpx.AsyncClient()
        
        async def fetch_one(url):
            try:
                resp = await self.http_client.get(url, timeout=30)
                return {"url": url, "content": resp.text}
            except Exception as e:
                return {"url": url, "error": str(e)}
        
        # 并发限流
        sem = asyncio.Semaphore(5)
        async def bounded_fetch(url):
            async with sem:
                return await fetch_one(url)
        
        tasks = [bounded_fetch(url) for url in sources]
        return await asyncio.gather(*tasks, return_exceptions=True)
    
    async def _summarize_documents(self, docs: List[Dict]):
        """并行总结文档（使用 LLM API）"""
        sem = asyncio.Semaphore(3)  # API 并发限制
        
        async def summarize_one(doc):
            async with sem:
                # 调用 LLM API（假设有 async 客户端）
                summary = await self.llm_client.summarize(doc["content"])
                return {"doc_id": doc["id"], "summary": summary}
        
        tasks = [summarize_one(doc) for doc in docs]
        return await asyncio.gather(*tasks, return_exceptions=True)
    
    def learn_from_sources(self, sources: List[str]) -> str:
        """同步接口（工具注册用）"""
        from model_tools import _run_async
        
        async def pipeline():
            # 阶段 1: 并行抓取
            materials = await self._fetch_materials(sources)
            valid_docs = [m for m in materials if "content" in m]
            
            # 阶段 2: 并行总结
            summaries = await self._summarize_documents(valid_docs)
            
            # 阶段 3: 存储到向量数据库（可能是同步的）
            # 如果 vector_db 是同步的，在这里调用
            for summary in summaries:
                self.vector_db.add(summary)
            
            return {
                "fetched": len(materials),
                "processed": len(summaries),
                "summaries": summaries
            }
        
        result = _run_async(pipeline())
        return json.dumps(result, ensure_ascii=False)
```

**优势**：
- 抓取和总结阶段并行，速度快
- 使用 `_run_async` 自动处理上下文
- 工具接口保持同步，易于集成

```python
                    model="gpt-4",
                    messages=[
                        {"role": "system", "content": "Summarize this document."},
                        {"role": "user", "content": content}
                    ]
                )
                
                return {
                    "url": url,
                    "summary": summary.choices[0].message.content
                }
            except Exception as e:
                return {"url": url, "error": str(e)}
        
        # 并行处理所有源
        with ThreadPoolExecutor(max_workers=5) as executor:
            futures = {executor.submit(fetch_and_summarize, url): url for url in sources}
            results = []
            for future in as_completed(futures):
                results.append(future.result())
        
        return json.dumps({"results": results}, ensure_ascii=False)
```

**优势**：
- 代码简单，易于理解
- 无需改造现有同步代码
- 内置节流（max_workers）

## 十、关键设计原则总结

### 10.1 上下文隔离

- **task_id 是隔离的核心标识符**
- 每个子 Agent 有独立的 session_id
- 工具缓存按 task_id 分区

### 10.2 中断友好

- 所有长时间运行的并行操作使用短超时轮询
- 工作线程注册到父 Agent，支持中断传播
- 使用 `future.cancel()` 而非强制终止线程

### 10.3 Event Loop 管理

- 主线程使用持久化全局 loop
- 工作线程使用每线程持久化 loop
- 网关/异步上下文使用独立线程 + `asyncio.run()`

### 10.4 异常容错

- `asyncio.gather(..., return_exceptions=True)` 捕获所有异常
- 每个 future 单独 try/except
- 降级处理：失败的任务返回部分结果

### 10.5 资源管理

- 始终使用上下文管理器 (`with`)
- ThreadPoolExecutor 的 `shutdown(wait=False, cancel_futures=True)` 快速清理
- 缓存的 async 客户端绑定到持久化 loop

## 结论

Hermes-Agent 的并行执行架构展示了如何在复杂的 Agent 系统中优雅地组合 ThreadPoolExecutor 和 asyncio：

1. **ThreadPoolExecutor** 用于同步工具的并行执行和子 Agent 委托
2. **asyncio.gather** 用于单个工具内的高并发 I/O 操作
3. **_run_async** 提供上下文感知的同步-异步桥接
4. **task_id** 实现会话级别的完全隔离

对于 self-learning skill，推荐使用**混合模式**：异步抓取 + 异步总结，通过 `_run_async` 包装成同步工具接口，既能获得高性能，又能无缝集成到现有工具栈。
