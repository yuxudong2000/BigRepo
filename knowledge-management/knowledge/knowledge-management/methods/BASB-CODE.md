## 核心理念

> **我们的大脑是用来产生想法的，不是用来存储想法的。**

第二大脑是一个**外部数字知识库**，通过系统性的捕获和整理，将散落的知识转化为创造性输出。

---

## CODE 方法（四步流程）

### C — Capture（捕获）

**目标**：捕获值得保存的想法和洞见

**原则**：
- **像策展人一样思考**：不要被动接受信息流，主动筛选
- **只保留引起共鸣的**：凭直觉，不需要分析性决策
- **工具集**：稍后读（Pocket/Instapaper/Readwise Reader）、网页剪裁、书摘导出

**问自己**：
- 我总是反复回到哪些主题？
- 哪些知识对我现在的项目有价值？
- 哪些内容适合未来相互碰撞产生新想法？

### O — Organize（组织）

**目标**：按照可操作性组织笔记（不按主题！）

**关键洞见**：信息按主题组织（类似图书馆杜威系统）是错误的。应该按**项目和行动目标**组织。

**PARA 系统**（见下文详解）

**每周批量处理**：不要实时分类，每周一次处理收件箱，每次10-20条

### D — Distill（提炼）

**目标**：将笔记提炼为可操作的精华摘要

**渐进式总结（Progressive Summarization）**：
1. 保存原文中最佳段落（第1层）
2. 高亮最重要的句子（第2层）
3. 用自己的话写执行摘要（第3层）
4. 在顶部写一句话总结（第4层）

**核心原则**：为你的"未来自我"写笔记（笔记即时间旅行）

**机会性提炼**：每次接触笔记时添加一点价值，不要一次完成所有提炼

### E — Express（表达）

**目标**：将知识转化为真实世界的成果

**中间包（Intermediate Packets）**：
- 每个项目由若干"中间包"组成
- 每次工作结束时完成至少一个中间包
- 中间包可以被未来项目复用

**核心观点**：
- 第二大脑是工厂，不是仓库
- 信息只有在被使用时才真正变成知识
- 公开分享带来意外机会

## PARA 系统（组织框架）

**P — Projects（项目）**
- 有明确目标、有截止日期的短期工作
- 例：写一篇文章、准备一场演讲、完成一个设计

**A — Areas（领域）**
- 无截止日期、需长期维护的责任
- 例：健康、职业发展、人际关系、财务

**R — Resources（资源）**
- 未来可能有用的主题和兴趣
- 例：某项技术、某个行业、某种工作方法

**A — Archive（归档）**
- 以上三类中不再活跃的内容

**分类逻辑**：
```
新笔记进来 → 哪个项目用得上？
          → 哪个领域相关？
          → 哪个资源主题？
          → 归档
```

**关键洞见**：按可操作性分类，让系统为你工作，而非消耗你。

## BASB vs Zettelkasten 的区别

| 维度 | BASB/CODE | Zettelkasten |
|------|-----------|--------------|
| 核心目标 | 项目产出，实用 | 思维涌现，学术 |
| 组织逻辑 | 按项目/行动 | 按链接/涌现 |
| 笔记粒度 | 较大（文章/段落级） | 极小（单一思想） |
| 适合人群 | 知识工作者、内容创作者 | 研究者、深度思考者 |
| 工具倾向 | Notion/Evernote | Obsidian/Roam |

---

## 实践建议

1. **从干净的状态开始**：将现有所有文件移入"Archive + 今日日期"文件夹，开始新系统
2. **不要追求完美分类**：随时可以通过搜索找到任何东西
3. **批量处理，而非实时分类**：每周一次处理收件箱
4. **每次接触笔记时添加价值**：哪怕只是一个标题或一行高亮
