## 💡 核心概念速查

### Skill 的本质

```
Skill = 可复用的专业工作流 + 领域知识封装

价值 = 一致性 + 专业性 + 可进化性 + 可传播性
```

### 四大技能类型

```
┌─────────────────────────────────────────┐
│  元技能 (Meta-Skill)                     │
│  - 通用质量保障框架                       │
│  - 适用于所有任务，自动激活                │
│  - 示例：meta-execution                  │
└─────────────────────────────────────────┘
                   ↓
┌─────────────────────────────────────────┐
│  领域技能 (Domain-Skill)                 │
│  - 特定领域专业能力                       │
│  - 需要特定触发词                         │
│  - 示例：product-thinking, essence-insight│
└─────────────────────────────────────────┘
                   ↓
┌─────────────────────────────────────────┐
│  工具技能 (Tool-Skill)                   │
│  - 特定任务的执行流程                     │
│  - 明确的输入输出                         │
│  - 示例：session-learning                │
└─────────────────────────────────────────┘
                   ↓
┌─────────────────────────────────────────┐
│  方法论技能 (Methodology-Skill)          │
│  - 思维框架和分析方法                     │
│  - 提供决策指导                          │
│  - 示例：brainstorming, karpathy-guidelines│
└─────────────────────────────────────────┘
```

### 八条黄金法则

```
1. 清晰性：一个 Skill 只解决一类问题
2. 明确性：触发条件必须明确
3. 可执行：流程必须可执行
4. 简洁性：使用检查清单而非长文本
5. 防御性：显式声明禁止项
6. 预见性：提供反模式和常见陷阱
7. 版本化：版本化和变更记录
8. 度量化：性能和效果度量
```
