# 信息质量与验证框架

## 信号 vs 噪音四维判断

**信号特征**（满足≥2条即为信号）：
1. **原创性**: 来自原创源（官方博客、论文、一手报告）
2. **新增量**: 包含新数据/新观点/新产品
3. **多源共振**: ≥2个独立源同时关注同一事件
4. **格局影响**: 可能改变领域竞争格局

**噪音特征**（满足任一条即为噪音）：
- 重复已知信息
- 纯标题党/炒作
- 无新信息增量
- 无法溯源验证

## 信息质量三角

```
         新鲜度
        ╱      ╲
      ╱          ╲
   权威性 ———— 可达性
```

| 维度 | 标准 |
|------|------|
| **新鲜度** | 日报/快讯±7天内，深度调研≤3个月，竞争格局≤3个月 |
| **权威性** | P0官方博客/官网/原始论文 > P1权威媒体 > P2行业KOL > P3社区 |
| **可达性** | 链接可访问，内容与引用描述一致 |

## 交叉验证分级规则

| 信息类型 | 验证要求 | 示例 |
|----------|---------|------|
| 事实类 | ≥2个独立来源 | "XX公司完成融资" |
| 格局类 | ≥2来源 + 信源≤3月 | "三足鼎立格局" |
| 数据类 | 回溯到原始报告 | "准确率达95%" |
| 预测类 | 标注为预测+来源 | "预计2026年底..." |

**未能交叉验证的事实**：必须标注 "⚠️ 未经多源确认"

---

## 三病扫描

| 病症 | 症状 | 处理 |
|------|------|------|
| 过时病 | 信源发布日期过旧 | 淘汰或找更新来源 |
| 空洞病 | 链接指向目录页 | 找到具体文章链接 |
| 凑数病 | 无实质新信息 | 淘汰，宁缺毋滥 |

---

## 趋势追踪方法

```
1. 跨周期扫描：回顾最近1-2周记录
2. 频率统计：统计每个主题出现次数
3. 趋势分类：
   📈 持续升温: 频率逐周上升
   ⚡ 突发事件: 单日/周爆发（Z-Score>2.0）
   🔥 本周焦点: 本周最高频主题
   🆕 新兴话题: 首次出现但多源关注
   📉 热度回落: 频率下降
4. 触发深研：持续升温/突发事件 → 标记"待深研"
```

## 知识沉淀规范

### 多角度提取方法论

同一知识源按不同维度分别提取沉淀：

| 维度 | 提取角度 | 沉淀位置 |
|------|---------|---------|
| 技术 | 架构、模型、训练 | 知识库/技术/ |
| 产品 | 定位、功能、用户 | 知识库/产品/ |
| 商业 | 模式、融资、竞争 | 知识库/商业/ |
| 趋势 | 行业趋势、战略 | 知识库/趋势/ |

### 数据一致性铁律

| 解法 | 含义 | 示例 |
|------|------|------|
| 脚本化 | 用脚本保证一致性 | 自动生成索引 |
| 全量化 | 不做"精选展示" | 直接展示全部 |
| 联动化 | 数据源变更触发更新 | 知识→索引→展示 |

### 增量更新规则

- 在已有章节末尾追加新内容
- 标注更新时间和来源
- **不要覆盖原有内容**（历史有价值）
- 相关概念间建立交叉引用链接

## 信息源审计检查（每月）

| 维度 | 检查项 | 淘汰标准 |
|------|--------|---------|
| 准确性 | 过去一月信息准确率 | <80% |
| 原创性 | 是否产出原创内容 | 纯转载>70% |
| 时效性 | 更新频率 | 超过1个月未更新 |
| 价值密度 | 信号/噪音比 | <30% |

**配置保鲜检查**（P1强制）：
- 搜索关键词是否仍然有效？
- 追踪名单是否与领域实际状态一致？
- 来源优先级是否需要调整？

## 洞察就绪度检查

### 定量指标

| 检查项 | 最低要求 |
|--------|---------|
| 知识库文档总数 | ≥10篇 |
| 核心概念文档数 | ≥3篇 |
| 主要玩家文档数 | ≥5篇 |
| 深度调研总数 | ≥3篇 |
| 信息源总数 | ≥5个 |

### 定性指标

能否清晰回答以下问题（至少4/5）：
1. 这个领域的核心概念是什么？
2. 主要玩家有哪些？各自定位是什么？
3. 当前的主流技术/方法是什么？
4. 近期有什么重要趋势/事件？
5. 这个领域的核心矛盾/问题是什么？

## 实战经验沉淀

### Gap-Driven Discovery（间隙驱动发现法）

信息源补充应该先绘制覆盖地图，**识别结构性空白**，然后**系统性补全**。

```
Step 1: 绘制现有覆盖地图 → 列出已覆盖的维度/品类/区域
Step 2: 识别结构性空白 → 系统性盲区 = 整个品类/维度 = 0 覆盖
Step 3: 用空白区域作为搜索框架 → 针对每个盲区设计搜索策略
```

### 多维渠道三角验证

验证实体信息时，**至少用三条独立路径交叉验证**：
- 路径A: 产品名/公司名 → 官方网站 → 确认定位、功能
- 路径B: 产品名 + "GitHub" → 代码仓库 → 确认技术栈、活跃度
- 路径C: 公司名 + "社交平台" → 社交渠道 → 确认运营状态

### 企业案例学习四阶段法

| 阶段 | 内容 |
|------|------|
| 材料准备 | 获取全部文档列表 → 完整提取 → 分类标注 |
| 结构化阅读 | 脊柱文档(全局框架) → 方法案例 → 平台度量 → 组织思想 |
| 知识合成 | 结构从内容中生长 + 跨文档连接 > 单文档摘要 |
| 验证输出 | 不看文档用自己的话讲清楚体系 |
