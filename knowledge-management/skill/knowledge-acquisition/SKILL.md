## 知识库位置

**按优先级自动检测：**

```
IF {project}/.knowledge/ 存在 → 使用项目知识库
ELSE IF ~/.knowledge/ 存在 → 使用用户知识库
ELSE → 询问用户初始化位置（项目/.knowledge/ | ~/.knowledge/ | 自定义）
```

> 🚨 **强制执行规则：检测必须通过 `list_files` 工具实际执行，禁止假设或推断。**
> 必须先 `list_files("{project}")` 确认 `.knowledge/` 目录是否存在，
> 再 `list_files("~")` 确认 `~/.knowledge/` 是否存在，
> 然后才能决定使用哪个路径。**跳过检测直接创建 = 严重错误。**

### 位置标记文件（快速判断）

知识库初始化后，`init_knowledge_base.py` 会在本技能目录写入一个标记文件：

| 文件名 | 含义 |
|--------|------|
| `.knowledge-base-in-project` | 知识库位于当前项目目录下（如 `{project}/.knowledge/`） |
| `.knowledge-base-in-personal` | 知识库位于个人主目录下（`~/.knowledge/`） |

文件内容是知识库的绝对路径。如果该标记文件存在，可直接读取以跳过 `list_files` 检测步骤；若不存在，则必须执行 `list_files` 检测。

## 📊 五阶主流程

### 第一阶：探源 (Source Discovery)

> **P0原则：信息源优先，不图快**

| 步骤 | 内容 |
|------|------|
| L1 广扫 | search_web + 多平台搜索，中英文多角度 |
| L2 精研 | fetch_web 全文提取 + 空白区补盲 |
| 质量评估 | 评估原创性、权威性、更新频率 |
| 覆盖分析 | 绘制信息源地图，识别盲区 |

**质量标准**：信息源数量 ≥15个，原创源比例 ≥50%，至少覆盖4种类型

### 第二阶：吸收 (Daily Absorption)

> 从信息源提取增量信息，区分噪音与信号

**信号特征**（满足≥2条即为信号）：
1. 原创性：来自原创源（官方博客、论文、一手报告）
2. 新增量：包含新数据/新观点/新产品
3. 多源共振：≥2个独立源同时关注
4. 格局影响：可能改变领域竞争格局

**信息质量三角**：
- 新鲜度：日报±7天内，深度调研≤3个月
- 权威性：P0官方 > P1权威媒体 > P2行业KOL > P3社区
- 可达性：链接可访问，内容与描述一致

### 第三阶：沉淀 (Knowledge Crystallization)

> 把零散信息结构化为知识

**核心原则**：
1. **多角度提取**：同一知识源按不同维度（技术/产品/商业/趋势）分别沉淀
2. **数据一致性铁律**：脚本化 + 全量化 + 联动化
3. **增量更新**：追加不覆盖，标注更新时间和来源
4. **P0: 新增文档必须更新索引**

### 第四阶：深潜 (Deep Research)

> 对重要主题进行深度调研

**触发条件**：
- 某主题出现≥3次/周
- 多个独立源同时关注
- 趋势标记为"📈持续升温"或"⚡突发事件"

**执行步骤**：
1. 确定调研范围（核心问题、边界、预期输出）
2. 多源信息收集（P0→P1→P2→P3优先级）
3. 交叉验证（事实类≥2独立源，数据类回溯原始报告）
4. 深度分析（追问"为什么"）
5. 形成结论 → 触发知识沉淀

### 第五阶：洞察 (Essence Insight)

> 发动本质洞察能力

**就绪条件**：知识库≥10篇 + 深度调研≥3篇

**五层分析框架**：
1. **现象层**：发生了什么
2. **因果层**：为什么发生
3. **模式层**：是否存在重复模式
4. **本质层**：底层规律是什么
5. **预测层**：未来可能如何演变

## 🚀 快速启动：新领域进入协议

```
Step 1: 领域初始化 → 确认领域边界 + 学习目标 + 创建知识库目录
Step 2: 信息源大规模探索（投入最多时间）→ L1广扫 + L2精研 + 覆盖分析
Step 3: 第一轮吸收 → 形成领域初印象 + 标记待深研主题
Step 4: 建立学习节奏 → 日报/周报频率 + 深研触发条件 + 进度检查点
```

## ⚠️ P0 强制规则

1. **内容完整性优先**：禁止API截断方式，必须完整提取
2. **批量提取前验证首篇完整性**
3. **图片是知识的一等公民**，禁止丢弃
4. **提取后执行覆盖度检查**
