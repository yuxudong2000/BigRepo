# 知识优化（Knowledge Optimization）

Agent的元能力——对知识库进行诊断、整理和优化。
**支持完整的诊断→决策→执行闭环**。

## 触发场景

- 用户说"检查知识库"、"知识库健康度"
- 用户说"整理知识"、"知识库乱了"
- 知识文档数量超过30个时（建议执行）
- 发现多个文档未被索引或过时时
- 每月一次深度Review

---

## 核心理念

> **知识不是存了就好，而是能被找到、被引用、被使用**
> **好的知识库 = 有索引 + 有分类 + 有引用 + 不重复 + 不过时**

## 知识库结构（动态获取）

> ⚠️ 知识库结构是动态的，**禁止硬编码**。每次使用前需要扫描获取。

### 知识库自动检测与加载

**按优先级自动检测并加载：**

| 优先级 | 路径 | 说明 |
|--------|------|------|
| 1 | `{project}/.knowledge/` | 当前项目的知识库（优先） |
| 2 | `~/.knowledge/` | 用户级知识库（备选） |

**检测逻辑：**

```
IF {project}/.knowledge/ 存在 → 使用项目知识库
ELSE IF ~/.knowledge/ 存在 → 使用用户知识库
ELSE → 询问用户是否初始化，选择存放位置：
  1. 当前项目 {project}/.knowledge/（推荐）
  2. 用户目录 ~/.knowledge/
  3. 自定义路径
```

> 🚨 **强制执行规则：检测必须通过 `list_files` 工具实际执行，禁止假设或推断。**
> 必须先 `list_files("{project}")` 确认 `.knowledge/` 目录是否存在，
> 再 `list_files("~")` 确认 `~/.knowledge/` 是否存在，
> 然后才能决定使用哪个路径。**跳过检测直接创建 = 严重错误。**

**用户强制重新初始化：** 用户说"重新初始化知识库"时，忽略检测，直接询问选择。

### 初始化命令

```bash
# KB_PATH 由用户选择决定
# 初始化后会在技能目录下生成 .knowledge-base-location 标记文件，记录知识库路径
python3 scripts/init_knowledge_base.py "${KB_PATH}"
```

### 获取当前结构

```bash
# 扫描知识库目录结构
python3 scripts/scan_knowledge_base.py "${KB_PATH}"
```

```
┌────────────────────────────────────────────────────────────────┐

```

## Phase 2: 健康度诊断

### 2.1 五项健康度检查

| 检查项       | 阈值            | 问题类型    | 优先级 | 说明              |
| :----------- | :-------------- | :---------- | :----: | :---------------- |
| **索引缺失** | 不在任何index中 | `orphan`    |   P0   | 无法被发现的知识  |
| **重复文档** | 标题相似>0.7    | `duplicate` |   P1   | 浪费空间/维护成本 |
| **分类混乱** | 位置与内容不符  | `misplaced` |   P1   | 难以查找          |
| **过时**     | >180天未更新    | `outdated`  |   P2   | 可能已失效        |
| **无引用**   | 无记忆/技能引用 | `isolated`  |   P2   | 可能已废弃        |

### 2.2 索引完整性检查

使用内嵌脚本检查：

```bash
python3 scripts/check_index_completeness.py "${KB_PATH}"
```

**输出示例**：

## Phase 2.5: 节点深度审计

> Phase 2 检查的是知识文档**之间**的关系（索引缺失、重复、分类混乱）；
> Phase 2.5 检查的是单篇知识文档**内部**的内容质量（时效性、一致性、完整性）。

### 2.5.1 筛选逻辑

| 信号                 | 阈值              | 权重 | 说明                   |
| :------------------- | :---------------- | :--: | :--------------------- |
| 文档行数             | > 200行           |  高  | 长文档更可能含结构问题 |
| 含统计数据/排名/数字 | 存在具体数值      |  高  | 数据易过时             |
| 含外部链接           | > 3个             |  高  | 链接腐烂风险           |
| 最后修改             | > 120天           |  中  | 可能内容已过时         |
| 无内部引用           | 记忆/技能均不引用 |  中  | 可能已废弃             |

### 2.5.2 五项审计检查

| 序号 | 检查项         | 问题类型               | 优先级 |
| :--: | :------------- | :--------------------- | :----: |
|  K1  | **信息时效性** | `info_outdated`        |   P0   |
|  K2  | **内部一致性** | `internal_conflict`    |   P0   |
|  K3  | **结构完整性** | `incomplete_structure` |   P1   |
|  K4  | **链接有效性** | `broken_link`          |   P1   |
|  K5  | **可读性评估** | `poor_readability`     |   P2   |

## Phase 3: 整理优化

### 3.1 用户确认

```
📋 知识库优化计划
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

发现以下问题需要处理：

🔴 P0 问题 (2项):
  1. [orphan] research/draft-report.md — 未被索引
  2. [orphan] notes/temp.md — 未被索引

🟡 P1 问题 (1项):
  1. [misplaced] guides/ai-research.md — 应属于 research/

🟢 P2 问题 (3项):
  1. [outdated] archives/old-guide.md — 200天未更新
  ...

是否执行优化？ [全部 / 仅P0 / 选择 / 跳过]
```

### 3.2 分类修正

| 原位置   | 内容特征        | 建议位置    |
| :------- | :-------------- | :---------- |
| 根目录   | 含"调研"/"分析" | `research/` |
| 根目录   | 含"指南"/"实践" | `guides/`   |
| 错误分类 | 根据内容判断    | 正确分类    |

### 3.3 过时标记

在文档顶部添加过时警告：

```markdown
> ⚠️ **本文档已过时** — 最后更新于 {DATE}，内容可能已失效。
> 如需引用，请先核实信息准确性。
```
