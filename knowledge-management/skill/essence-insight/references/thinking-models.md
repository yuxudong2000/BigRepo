## 核心思维模型（7个）

### 1. 五层思考框架

**定义**：从执行到战略的层层递进分析
**适用场景**：需要深度理解某件事的本质时

```
执行层：怎么做？（方法/步骤）
    ↓
方法层：为什么这样做？（原理/依据）
    ↓
系统层：这是整体的哪部分？（位置/关系）
    ↓
规律层：背后有什么规律？（模式/趋势）
    ↓
哲学层：更本质的是什么？（第一性原理）
```

**使用标志**：每层追问"为什么"，直到触及本质

---

### 2. 本质洞察

**定义**：穿透表象看本质，抽丝剥茧找底层规律
**适用场景**：分析产品、技术、趋势的深层逻辑

```
洞察事情（人造物） → 追问"为什么这样设计"
洞察对象（人/企业） → 追问"底层动机是什么"
```

**核心方法**：

- 洞察闭环五层框架：L1表象→L2规律+类比→L3验证→L4演变→L5预测
- 类比是本质洞察的"试金石"——真正理解本质后一定能找到好类比

**使用标志**：输出包含形象类比和可验证的预测

---

### 3. 类比驱动

**定义**：通过类比将复杂概念映射到熟悉场景
**适用场景**：解释复杂事物、验证理解深度

```
核心结构提炼 → 熟悉场景搜索 → 结构匹配验证 → 简化表达
```

**好类比标准**：

- 受众熟悉
- 结构相似
- 简洁生动
- 有启发性

**使用标志**：用"就像...一样"让人秒懂

---

### 4. 隐藏前提识别

**定义**：任何结论都建立在前提之上，找到并验证隐藏前提
**适用场景**：评估方案可行性、验证信息准确性

```
结论 → 这个结论成立的前提是什么？
      → 这些前提在当前情境下成立吗？
      → 有没有被忽略的重要前提？
```

**使用标志**：主动质疑"这个前提可靠吗"

---

### 5. 产品思维

**定义**：以用户为中心，顺用户的意而非按我的来
**适用场景**：所有交付物（网页/文档/图片/PPT/报告）

```
核心四问：
1. 谁在用这个？（用户画像）
2. 什么场景用？（使用情境）
3. 用户要什么？（核心需求）
4. 怎么让他爽？（体验优化）
```

**使用标志**：交付前切换到用户视角自检

---

### 6. 系统思维

**定义**：将事物视为相互关联的系统，关注整体而非局部
**适用场景**：理解复杂问题、设计技术架构

```
要素分析：系统由哪些要素组成？
关系分析：要素之间如何相互作用？
边界分析：系统的边界在哪？与外部如何交互？
动态分析：系统如何随时间演变？
```

**使用标志**：画出要素关系图，识别关键连接点

---

### 7. 批判性思维

**定义**：对信息进行审慎评估，区分事实与观点
**适用场景**：信息过滤、决策评估、信息源评价

```
信息四问：
1. 这是事实还是观点？
2. 信息来源可靠吗？（一手/二手/三手）
3. 有没有利益相关？
4. 有没有反例或对立观点？
```

**使用标志**：标注信息的置信度

## 候选思维模型（4个）

> 这些模型正在观察中，如果使用频率够高且效果明显，将升级为核心模型

### C1. 第一性原理

**定义**：回到问题的最基本假设，从零开始推理
**当前状态**：已部分整合到五层思考框架的哲学层

### C2. 逆向思维

**定义**：从目标/结果反推起始条件
**当前状态**：在方案设计中偶尔使用

### C3. 概率思维

**定义**：用概率而非确定性来思考不确定事件
**当前状态**：在风险评估中使用

### C4. 复利思维

**定义**：关注积累性收益，优先选择有复利效应的行动
**当前状态**：已整合到小无相功的复利循环
