# 通用踩坑模式汇总

> 从实践中提炼的常见执行陷阱，每个模式都有明确的识别信号和防护策略。

---

## 1. 静默失败模式

### 问题描述

在多步骤工作流中，某一步失败但被忽略，继续执行后续步骤，导致最终结果错误但难以追溯根因。

### 典型场景

- 命令执行返回错误但未检查
- API调用失败但直接跳过
- 文件操作异常但继续处理

### 防护策略

```
关键步骤执行后，必须检查返回值/状态：
- 命令执行：检查退出码和输出内容
- API调用：检查响应状态码和错误信息
- 文件操作：验证文件确实存在/已修改

发现异常时：停下来报告，而非静默继续
```

### 自检信号

当你发现自己在"跳过错误继续执行"时，停下来问：这个错误会不会在以后变成更大的问题？答案通常是"会"。

## 2. 假完成模式

### 问题描述

声称任务完成，但实际上核心工作未执行，或执行了但产出无效。

### 典型场景

- 状态标记为"完成"但实际产出为空
- 验证文件存在≠验证工作完成（文件可能早就存在）
- 竞态条件：任务刚发送就轮询状态，获得"未运行"误判为"已完成"

### 防护策略

```
完成判定三原则：
1. 验证实际产出（而非状态标记）
2. 检查产出时间（是否在本次执行期间生成）
3. 验证产出内容（是否符合预期格式和要求）

状态轮询规则：
- 任务发送后等待足够时间再轮询
- 连续多次确认状态一致才认为完成
- 验证产出消息数量和内容
```

## 3. 清单即全集谬误

### 问题描述

认为现有清单是完整的，直接基于清单执行批量操作，结果遗漏大量对象。

### 典型场景

- 给N个文件批量添加某内容，只看了现有清单，实际应处理的文件数是清单的两倍
- 基于文档中的列表执行操作，但文档已过时

### 防护策略

```
批量操作前必做：
1. 先全量扫描确认完整集合（用搜索/遍历，而非信任清单）
2. 扫描结果数 vs 清单数，有差异必须解释清楚
3. 刚写的清单 ≠ 完整清单，遗漏率可达50%
```

## 4. 声明即执行谬误

### 问题描述

在文档/配置中声明了某条规则或要求，就认为它会被自动遵守，实际上执行流程中没有对应步骤。

### 典型场景

- 配置文件中标记 `requires_xxx: true`，但流程中从未检查此标记
- README中写了"执行前请先XXX"，但没人真的执行
- 注释中写了约束条件，但代码没有实现

### 防护策略

```
规则生效三要素：
1. 声明层：规则写在哪里
2. 检测层：谁来检查是否符合
3. 执行层：不符合时怎么处理

只有声明层而没有检测层和执行层 = 规则不存在
```

## 5. 元工具自举盲区

### 问题描述

用来管理其他东西的工具/规则，无法有效管理自己，导致元工具本身出现问题但无人发现。

### 典型场景

- 质量检查规范本身有逻辑漏洞
- 审计工具的审计规则有错误
- 规范文档中的规则编号重复/矛盾

### 防护策略

```
修改元工具/元规则后：
1. 必须做外部审计（找矛盾、找遗漏、验一致性）
2. 不能只靠自己Review——自己审计自己有天然盲区
3. 如果包含清单/规则/标准，验证"规则条件"与"清单数据"的逻辑一致性
```

### 类比

审计事务所也需要被审计——鞋匠的孩子不能没鞋穿。

## 6. 中途中断模式

### 问题描述

任务执行中途遇到阻碍，直接退出，未完成必要的收尾工作，导致状态混乱、进度丢失。

### 防护策略

```
无论任务是否遇到阻碍，退出前必须完成：
1. 记录当前状态和已完成的工作
2. 标注遇到的问题和待处理事项
3. 确保可以从断点恢复
```

---

## 7. 长流程截断模式

### 问题描述

多步骤任务在中间某步骤卡住，前半段已完成的工作没有阶段性产出，导致价值丢失。

### 防护策略

```
长流程任务规则：
1. 每完成2-3个步骤，立即输出阶段性结果
2. 即使后续步骤失败，前面阶段的成果可独立使用
3. 记录当前进度，支持从失败步骤继续
4. 5步以上长流程任务，强制每2-3步产出阶段性文件
```

## 快速参考

| 模式 | 一句话识别 | 一句话防护 |
|:-----|:-----------|:-----------|
| 静默失败 | 跳过错误继续执行 | 每步检查，异常即停 |
| 假完成 | 有状态无产出 | 验产出不验状态 |
| 清单即全集 | 信任现有列表 | 操作前全量扫描 |
| 声明即执行 | 写了规则就安心 | 规则要有执行步骤 |
| 元工具盲区 | 自己审自己 | 修改后外部审计 |
| 中途中断 | 遇阻直接退 | 退出前必须收尾 |
| 长流程截断 | 前功尽弃 | 2-3步一个检查点 |
