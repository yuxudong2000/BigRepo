# 场景分析检查清单

在开始任何页面/界面设计前，使用此清单进行场景分析。

---

## 第一步：识别使用场景

### 时间维度

- [ ] 用户什么时候打开这个页面？
  - 早上第一件事（速览模式）
  - 工作中随时（工具模式）
  - 定期回顾（分析模式）
  - 向他人展示（演示模式）

### 设备维度

- [ ] 用户用什么设备？
  - 手机（单手操作、竖屏、小屏幕）
  - 平板（触摸、中屏幕）
  - PC（鼠标键盘、大屏幕、多窗口）

### 时长维度

- [ ] 用户停留多久？
  - 30秒速览：核心数据前置、变化高亮
  - 5分钟浏览：信息层级清晰、支持快速跳转
  - 深入分析：完整数据、支持筛选钻取

### 频率维度

- [ ] 使用频率如何？
  - 每日使用：强调"变化"和"新增"
  - 每周使用：强调"周期对比"和"趋势"
  - 偶尔使用：强调"全貌"和"导航"

---

## 第二步：定义用户画像

### 用户预期

- [ ] 用户打开页面时，心里想的是什么？
- [ ] 用户期望看到什么？
- [ ] 用户不想看到什么？

### 用户任务

- [ ] 用户想完成什么核心任务？
- [ ] 任务完成的标志是什么？
- [ ] 任务失败会是什么情况？

### 用户背景

- [ ] 用户的技术背景如何？
- [ ] 用户对这类信息的熟悉程度？
- [ ] 用户的审美偏好？

---

## 第三步：识别欲望/痛点

### 满足感（爽点）

用户想获得什么"爽"的感觉？

- [ ] 信息获取：一眼看到关键信息
- [ ] 成就感：看到成长和进步
- [ ] 掌控感：全貌尽在掌握
- [ ] 惊喜感：发现有趣的洞察
- [ ] 炫耀感：可以展示给别人看

### 恐惧感（痛点）

用户想消除什么焦虑/担忧？

- [ ] 遗漏恐惧：担心漏掉重要信息
- [ ] 复杂恐惧：担心看不懂、操作复杂
- [ ] 时间恐惧：担心浪费时间
- [ ] 错误恐惧：担心做出错误判断

---

## 第四步：做出设计决策

### 信息优先级

- [ ] 必须第一眼看到的信息是什么？
- [ ] 可以折叠/收起的信息是什么？
- [ ] 可以完全隐藏的信息是什么？

### 默认状态

- [ ] 页面加载后的默认视图是什么？
- [ ] 默认选中/展开的内容是什么？
- [ ] 默认时间范围/筛选条件是什么？

### 交互路径

- [ ] 用户完成核心任务需要几次点击？
- [ ] 能否减少点击次数？
- [ ] 用户迷路的可能性有多大？

### 视觉重点

- [ ] 什么元素应该最突出？
- [ ] 什么元素应该弱化？
- [ ] 视觉层级是否支持任务完成？

---

## 输出模板

完成场景分析后，用以下格式输出：

```markdown
## 场景分析结果

### 场景

- 使用时机：[描述]
- 使用设备：[手机/PC/平板]
- 停留时长：[30秒/5分钟/深入分析]
- 使用频率：[每日/每周/偶尔]

### 用户画像

- 预期：[用户打开页面期望什么]
- 任务：[用户想完成什么]
- 背景：[技术水平/熟悉程度]

### 欲望/痛点

- 满足感：[用户想要的爽点]
- 恐惧感：[用户想消除的痛点]

### 设计决策

- 优先展示：[列表]
- 折叠/弱化：[列表]
- 隐藏：[列表]
- 默认视图：[描述]
- 交互路径：[核心任务的点击次数]
```

---

## 快速自检

在开始编码前，确认可以回答以下问题：

1. ✅ 我知道用户在什么场景下使用这个页面
2. ✅ 我知道用户想完成什么任务
3. ✅ 我知道用户想要的满足感或想消除的恐惧
4. ✅ 我的设计决策是基于以上分析，而非"我觉得这样酷"
