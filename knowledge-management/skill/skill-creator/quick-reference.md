# Skill 创建快速参考

> 一页纸速查手册

---

## 🎯 三步快速创建

```
1. RED   → 观察 AI 失败（基线测试）
2. GREEN → 编写 Skill（让测试通过）
3. REFACTOR → 查漏补缺（3-5轮迭代）
```

---

## ✅ 创建前检查清单

- [ ] 回答了10问清单前5个问题
- [ ] 运行了基线测试（不加载Skill）
- [ ] 记录了AI的错误行为
- [ ] 识别了根本原因

## 📋 必填字段（Frontmatter）

```yaml
---
name: skill-name              # kebab-case，无特殊字符
description: Use when [...]   # 只描述触发条件，不总结工作流！
version: 1.0.0               # 语义化版本号
---
```

---

## 🚨 Skill 必备元素

```markdown
## 触发场景
- ✅ 适用场景
- ❌ 不适用场景

## P0 强制规则
- ❌ 禁止：[...]
- ✅ 必须：[...]

## 核心流程/检查清单
- [ ] 检查项1
- [ ] 检查项2

## 常见错误
❌ 错误1
**症状**：[...]
**修复**：[...]
```

---

## 💎 Description 黄金法则

```yaml
# ❌ 错误：总结工作流
description: Use when creating skills - write test, then skill, then verify

# ✅ 正确：只描述触发条件
description: Use when creating new skills or editing existing skills
```

**为什么？**
- AI 会跟随 description 而不读完整 Skill
- Description 应该回答"何时用"，不是"怎么用"

## ⚠️ 常见错误

| 错误 | 症状 | 修复 |
|------|------|------|
| **先写后测** | 写完才想起测试 | 删除重来 |
| **模糊词汇** | "可能"、"建议" | 改为"必须"、"禁止" |
| **缺少检查** | AI跳过步骤 | 添加清单 |
| **过度抽象** | 不知何时用 | 具体化触发场景 |

## 🧪 测试技巧

### 基线测试
```
use_subagent(
    subagent_name="baseline",
    task="[场景] 约束：不加载任何skill",
    background=false
)
```

### Green 测试
```
use_subagent(
    subagent_name="with-skill",
    task="[场景] 要求：加载skill: [name]",
    background=false
)
```

### 验证标准
- ✅ AI 不再犯基线测试的错误
- ✅ AI 按 Skill 步骤执行
- ✅ 输出符合预期

## 💡 核心洞察

```
Skill = 认知压缩 + 知识结晶 + 执行引擎

从气态（零散知识）
  ↓ 凝聚
到液态（结构化文档）
  ↓ 结晶  
到固态（可执行Skill）
```

---

## 🎯 记住这3条

1. **先测试，后编写**（铁律，无例外）
2. **Description = 何时用**（不是怎么用）
3. **从简单开始**（1.0 → 2.0 → 3.0 渐进）
